# Cybersecurity

## Core Security Concepts
CIA triad: Confidentiality ensures information is accessible only to authorized parties. Integrity ensures information is accurate and has not been tampered with. Availability ensures systems and data are accessible when needed. Authentication verifies who you are using passwords biometrics or tokens. Authorization determines what you are allowed to do after authentication. The principle of least privilege grants users and programs only the minimum permissions needed to perform their function. Defense in depth uses multiple layers of security so that if one fails others still protect. Security through obscurity alone is not sufficient but can be one layer of defense.

## Cryptography
Symmetric encryption uses the same key for encryption and decryption. It is fast and suitable for large data. Examples: AES Advanced Encryption Standard is the industry standard using 128 256-bit keys. The key distribution problem: both parties must securely share the key. Asymmetric encryption uses a public key to encrypt and a private key to decrypt. Solves the key distribution problem. RSA is the most common asymmetric algorithm. Much slower than symmetric encryption. Used for key exchange and digital signatures. TLS uses asymmetric encryption to exchange a symmetric session key then encrypts data with that faster symmetric key. Hashing converts data to a fixed-length digest. It is one-way and cannot be reversed. SHA-256 is commonly used for integrity verification and password storage. Salting adds random data to passwords before hashing to prevent rainbow table attacks.

## Common Attacks
SQL injection inserts malicious SQL into input fields to manipulate database queries. Prevented by using parameterized queries or prepared statements never string concatenation. Cross-site scripting XSS injects malicious scripts into web pages viewed by other users. Prevented by escaping output and using Content Security Policy headers. Cross-site request forgery CSRF tricks users into submitting requests they did not intend by exploiting their authenticated session. Prevented using CSRF tokens. Man-in-the-middle MITM attack intercepts communication between two parties. Prevented by TLS encryption and certificate verification. Denial of service DoS overwhelms a system with traffic making it unavailable. DDoS uses multiple sources. Phishing tricks users into revealing credentials through fake emails or websites. Brute force tries all possible passwords. Prevented with rate limiting account lockouts and strong passwords.

## Web Security
HTTPS encrypts HTTP traffic using TLS protecting confidentiality and integrity. All production websites should use HTTPS. Same-origin policy prevents JavaScript on one origin from accessing resources on another origin. CORS Cross-Origin Resource Sharing allows controlled exceptions to same-origin policy using HTTP headers. JWT JSON Web Token is a compact self-contained token for transmitting claims between parties. Consists of header payload and signature. Signed but not encrypted by default so do not store sensitive data in payload. OAuth 2.0 is an authorization framework allowing third-party applications to access user resources without exposing credentials. OpenID Connect is an authentication layer on top of OAuth 2.0. Cookies should have HttpOnly flag preventing JavaScript access and Secure flag requiring HTTPS.

## Authentication and Passwords
Never store passwords in plain text. Store a salted hash using bcrypt scrypt or Argon2. These are slow by design making brute force expensive. Multi-factor authentication MFA requires something you know like password, something you have like phone, or something you are like fingerprint. Time-based one-time passwords TOTP generate a new 6-digit code every 30 seconds using a shared secret. Used in apps like Google Authenticator. Session management: after login issue a session token stored in a cookie. Invalidate sessions on logout. Session fixation attack sets a known session ID before login. Prevent by regenerating session ID after authentication. Rate limiting prevents brute force attacks by limiting login attempts per time period.
