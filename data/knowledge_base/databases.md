# Databases

## SQL Fundamentals
SQL Structured Query Language is used to interact with relational databases. SELECT retrieves data. INSERT adds rows. UPDATE modifies rows. DELETE removes rows. WHERE filters rows. ORDER BY sorts results. GROUP BY groups rows for aggregate functions like COUNT, SUM, AVG, MAX, MIN. HAVING filters groups after GROUP BY. JOIN combines rows from multiple tables. INNER JOIN returns rows with matching values in both tables. LEFT JOIN returns all rows from the left table and matching rows from the right, with NULL for no match. RIGHT JOIN is the reverse. FULL OUTER JOIN returns all rows from both tables. CROSS JOIN returns the Cartesian product. Self join joins a table with itself. Subqueries are queries nested inside another query. Correlated subqueries reference the outer query. EXISTS checks if a subquery returns any rows. IN checks if a value is in a subquery result. DISTINCT removes duplicate rows. LIMIT restricts the number of rows returned.

## Normalization
Normalization organizes a database to reduce redundancy and improve data integrity by decomposing tables. First Normal Form 1NF: each column contains atomic indivisible values and each row is unique. No repeating groups or arrays. Second Normal Form 2NF: must be in 1NF and every non-key attribute must fully depend on the entire primary key not just part of it. Eliminates partial dependencies. Applies only to tables with composite primary keys. Third Normal Form 3NF: must be in 2NF and no non-key attribute should depend on another non-key attribute. Eliminates transitive dependencies. Boyce-Codd Normal Form BCNF: every determinant must be a candidate key. Stricter than 3NF. Denormalization intentionally introduces redundancy to improve read performance in data warehouses by reducing joins.

## Indexing
An index is a data structure that speeds up data retrieval at the cost of additional write and storage overhead. B-tree index is the default type in most databases and supports equality and range queries. Clustered index determines the physical order of rows in the table. There can be only one per table. Non-clustered index stores pointers to the actual rows and a table can have multiple. Composite index covers multiple columns. The order of columns matters. The leftmost prefix rule means a composite index on (A, B, C) can be used for queries on A, on (A, B), or on (A, B, C) but not on (B) or (C) alone. Hash index supports only equality queries not range queries and is faster for exact match lookups. Index selectivity is the ratio of unique values to total rows. High selectivity means fewer rows per value and a more useful index. Full table scans occur when the optimizer decides an index is not selective enough or the query does not use the index correctly.

## Transactions and ACID
A transaction is a sequence of operations treated as a single unit of work. ACID properties ensure reliable transactions. Atomicity: a transaction is all or nothing. If any part fails the entire transaction is rolled back. Consistency: a transaction brings the database from one valid state to another preserving all integrity constraints. Isolation: concurrent transactions execute as if they were sequential. One transaction should not see uncommitted changes of another. Durability: once a transaction is committed its changes are permanent even if the system crashes. Isolation levels from lowest to highest: Read Uncommitted allows dirty reads of uncommitted data. Read Committed prevents dirty reads. Repeatable Read prevents dirty and non-repeatable reads but allows phantom reads. Serializable the highest level prevents all anomalies but has the lowest concurrency. Locks: shared lock allows multiple readers. Exclusive lock allows only one writer. Two-phase locking acquires all locks before releasing any to ensure serializability.

## NoSQL Databases
NoSQL databases sacrifice some ACID properties for scalability and flexibility. Types: Document stores like MongoDB store data as JSON-like documents. Flexible schema allows different fields per document. Good for hierarchical data. Key-value stores like Redis store simple key-value pairs. Extremely fast for caching and session management. Column-family stores like Cassandra store data in columns grouped into families. Optimized for writes and time-series data. Graph databases like Neo4j store nodes and relationships. Optimized for traversing relationships. CAP theorem states that a distributed system can guarantee only two of three: Consistency all nodes see the same data at the same time, Availability every request receives a response, and Partition Tolerance the system continues despite network splits. In practice partition tolerance is necessary so systems choose between CP and AP. BASE: Basically Available Soft state Eventually consistent describes NoSQL systems as opposed to ACID.
