# Data Structures

## Arrays
An array stores elements in contiguous memory locations. Access by index is O(1). Search in unsorted array is O(n). Search in sorted array using binary search is O(log n). Insertion at the beginning or middle is O(n) because elements must shift. Insertion at the end is O(1) amortized for dynamic arrays. Arrays are cache-friendly due to contiguous memory storage. A 2D array stores elements in rows and columns accessed by row and column index in O(1).

## Linked Lists
A linked list stores elements as nodes where each node contains data and a pointer to the next node. Unlike arrays, linked lists do not require contiguous memory. Singly linked list: each node points to the next node. Traversal is O(n). Insertion at head is O(1). Insertion at tail is O(n) without a tail pointer, O(1) with one. Deletion of a node requires finding the previous node so it is O(n) in general. Doubly linked list: each node has pointers to both next and previous. Deletion of a known node is O(1). Insertion before a known node is O(1). Circular linked list: the tail points back to the head, useful for round-robin scheduling. Key comparison — linked list has O(1) insertion at head but O(n) random access. Array has O(1) random access but O(n) insertion and deletion.

## Stacks
A stack is a Last In First Out (LIFO) data structure. Operations: push adds to top in O(1), pop removes from top in O(1), peek views top without removing in O(1). Implemented using an array or linked list. Common uses: function call stack for recursion, undo operations in editors, balanced parentheses checking, iterative depth-first search DFS, expression evaluation using operator precedence, and browser back navigation. Stack overflow occurs when recursion exceeds the call stack size limit.

## Queues
A queue is a First In First Out (FIFO) data structure. Operations: enqueue adds to rear in O(1), dequeue removes from front in O(1), peek views front in O(1). Types of queues: simple queue is FIFO. Circular queue connects rear to front to reuse array space efficiently. Priority queue dequeues elements by priority not arrival order and is implemented with a heap where dequeue is O(log n) and enqueue is O(log n). Deque or double-ended queue allows insertion and deletion at both ends in O(1). Common uses: BFS graph traversal, CPU scheduling with round-robin, print spooling, and task scheduling.

## Trees
A tree is a hierarchical data structure. The root is the topmost node. Leaf nodes have no children. Height of a tree is the longest path from root to any leaf. Binary tree: each node has at most 2 children called left and right. Full binary tree: every node has 0 or 2 children. Complete binary tree: all levels filled except possibly the last which fills left to right. Perfect binary tree: all levels completely filled. Tree traversals: Inorder visits Left then Root then Right and gives sorted output for BST. Preorder visits Root then Left then Right and is useful for copying a tree. Postorder visits Left then Right then Root and is useful for deleting a tree. Level-order traversal uses a queue and visits nodes level by level.

## Binary Search Tree (BST)
A BST is a binary tree where every node satisfies: all values in left subtree are less than the node value and all values in right subtree are greater than the node value. Search is O(log n) average case and O(n) worst case on an unbalanced skewed tree. Insert is O(log n) average and O(n) worst case. Delete is O(log n) average and O(n) worst case. A skewed BST occurs when all insertions are in sorted order, making it degenerate into a linked list with O(n) operations. Balanced BSTs like AVL tree and Red-Black tree maintain O(log n) height by performing rotations on insertion and deletion. AVL tree maintains balance factor of at most 1. Red-Black tree is used in most standard library implementations including Java TreeMap and C++ map.

## Heaps
A heap is a complete binary tree satisfying the heap property. Min-heap: every parent is less than or equal to its children so the root is the minimum element. Max-heap: every parent is greater than or equal to its children so the root is the maximum. Heaps are implemented as arrays where for node at index i: left child is at 2i+1, right child is at 2i+2, parent is at (i-1) divided by 2. Operations: insert is O(log n) by adding at end then bubbling up. Extract min or max is O(log n) by swapping root with last element, removing last, then bubbling down. Peek at root is O(1). Build heap from unordered array is O(n) using bottom-up heapify which is faster than inserting elements one by one. Heap sort builds a max-heap in O(n) then repeatedly extracts the max giving O(n log n) total with O(1) extra space but is not stable.

## Hash Tables
A hash table stores key-value pairs. A hash function maps keys to array indices called buckets. Best case: O(1) insert, delete, and search. Collisions occur when two keys map to the same bucket. Collision resolution: chaining uses a linked list at each bucket giving O(n) worst case if all keys collide. Open addressing probes the next available slot using linear probing, quadratic probing, or double hashing. Load factor equals number of elements divided by number of buckets. When load factor exceeds 0.7 the table resizes by rehashing all elements which is O(n) but gives O(1) amortized per insert. A good hash function distributes keys uniformly. For strings a polynomial rolling hash is commonly used.

## Graphs
A graph consists of vertices and edges. Directed graph or digraph has directional edges. Undirected graph has bidirectional edges. Weighted graph assigns costs to edges. Representations: adjacency matrix is a V by V array where matrix[u][v] is 1 or the weight if edge exists. Edge check is O(1) but space is O(V squared) making it good for dense graphs. Adjacency list stores neighbors for each vertex. Space is O(V plus E). Edge check is O(degree of vertex). Good for sparse graphs. Graph traversals: BFS uses a queue and explores all neighbors before going deeper. It finds the shortest path in unweighted graphs and runs in O(V plus E). DFS uses a stack or recursion and explores as deep as possible before backtracking. It runs in O(V plus E) and is used for cycle detection, topological sort, and connected components. Shortest path algorithms: Dijkstra works on non-negative weights in O((V plus E) log V) with a priority queue. Bellman-Ford handles negative weights in O(VE). Floyd-Warshall finds all pairs shortest paths in O(V cubed). Minimum spanning tree algorithms: Kruskal sorts all edges and uses Union-Find in O(E log E). Prim is greedy using a priority queue in O((V plus E) log V). Topological sort gives a linear ordering of vertices in a DAG such that for every edge u to v, u comes before v. Used in dependency resolution and build systems. DFS-based topological sort runs in O(V plus E). Cycle detection in undirected graph uses DFS and checks for back edges. In directed graph use three-color DFS with white gray black states.
