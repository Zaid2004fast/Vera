# Operating Systems

## Processes and Threads
A process is an independent program in execution with its own memory space including code, data, heap, and stack segments. A thread is a unit of execution within a process. Threads within the same process share memory space and resources but have their own stack and registers. Creating a thread is faster than creating a process because no new memory space is needed. Context switching between threads is cheaper than between processes. A process can have multiple threads running concurrently. Advantages of threads include shared memory communication and lower overhead. Disadvantages include race conditions and the need for synchronization. Inter-process communication IPC includes pipes, message queues, shared memory, and sockets. Shared memory is the fastest IPC method. Process states: new, ready, running, waiting or blocked, terminated. The operating system manages process state transitions using the process control block PCB which stores process ID, state, program counter, CPU registers, and memory information.

## CPU Scheduling
CPU scheduling determines which process runs on the CPU at any time. First Come First Served FCFS is non-preemptive and can cause the convoy effect where short processes wait behind long ones. Average waiting time can be high. Shortest Job First SJF picks the process with the shortest burst time. It is optimal for minimizing average waiting time but requires knowing burst times in advance. Preemptive version is Shortest Remaining Time First SRTF. Round Robin RR assigns a fixed time quantum to each process in a circular queue. Preemptive and fair. If quantum is too large it behaves like FCFS. If too small, excessive context switching overhead occurs. Priority scheduling assigns priorities to processes and runs the highest priority first. Can cause starvation of low priority processes. Aging increases priority over time to prevent starvation. Multilevel Queue Scheduling divides processes into categories each with its own queue and scheduling algorithm. Multilevel Feedback Queue allows processes to move between queues based on behavior.

## Memory Management
Virtual memory allows processes to use more memory than physically available by storing parts of the process on disk. The page table maps virtual addresses to physical addresses. A page fault occurs when the required page is not in memory and must be loaded from disk. Demand paging loads pages only when needed. Page replacement algorithms decide which page to remove when memory is full. FIFO replaces the oldest page and can suffer from Belady's anomaly where adding more frames increases page faults. Optimal OPT replaces the page not needed for the longest time. It is not implementable in practice but serves as a benchmark. Least Recently Used LRU replaces the page not used for the longest time. Approximated with clock algorithm or second chance algorithm. Thrashing occurs when a process spends more time paging than executing because it does not have enough frames. The working set model tracks pages used in a recent time window to allocate sufficient frames. Segmentation divides memory into logical segments of variable size like code, data, and stack. Paging divides memory into fixed-size pages eliminating external fragmentation but causing internal fragmentation if the last page is not full.

## Synchronization and Deadlocks
A race condition occurs when multiple threads access shared data concurrently and the outcome depends on execution order. A critical section is code that accesses shared resources and must not execute concurrently. Mutual exclusion ensures only one thread is in the critical section at a time. Semaphores are integer variables used for synchronization. Binary semaphore is a mutex with values 0 and 1. Counting semaphore allows up to n threads. Wait or P decrements and blocks if zero. Signal or V increments and wakes a waiting thread. Mutex lock is owned by the thread that acquired it and only that thread can release it. Monitor is a high-level synchronization construct with mutual exclusion built in and condition variables for waiting. Deadlock occurs when two or more processes are each waiting for a resource held by the other. Four conditions for deadlock: mutual exclusion, hold and wait, no preemption, and circular wait. Deadlock prevention eliminates one of the four conditions. Deadlock avoidance uses Banker's algorithm to ensure the system stays in a safe state. Deadlock detection allows deadlock to occur then recovers by terminating processes or preempting resources.

## File Systems
A file system organizes and stores data on disk. Inode stores metadata about a file including size, permissions, timestamps, and pointers to data blocks but not the filename. Directory maps filenames to inodes. File allocation methods: contiguous allocation stores all blocks sequentially giving fast sequential and random access but causes external fragmentation. Linked allocation chains blocks together using pointers with no fragmentation but random access is O(n). Indexed allocation uses an index block with pointers to all data blocks giving fast random access with no external fragmentation. FAT file allocation table is a linked list stored in a table for efficiency. NTFS and ext4 are modern journaling file systems that log changes before committing to prevent corruption on crash. Disk scheduling algorithms: FCFS processes requests in order. SSTF shortest seek time first minimizes seek time but can cause starvation. SCAN elevator algorithm moves the disk arm in one direction servicing requests until it reaches the end then reverses. C-SCAN circular SCAN only services requests on the way in one direction then jumps back to the beginning.
