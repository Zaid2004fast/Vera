# Compilers and Programming Languages

## Compilation Phases
A compiler translates source code into machine code. Phases: Lexical analysis or scanning breaks source code into tokens such as keywords identifiers operators and literals. Syntax analysis or parsing builds a parse tree or abstract syntax tree AST from tokens according to the grammar rules of the language. Semantic analysis checks for type errors undeclared variables and other meaning-related errors using the AST and symbol table. Intermediate code generation produces language-independent intermediate representation like three-address code. Optimization improves intermediate code for speed or size by eliminating redundant computations, inlining functions, and loop optimizations. Code generation produces target machine code from optimized intermediate representation.

## Lexical Analysis
The lexer or scanner reads characters and groups them into tokens. A token is a categorized unit such as IDENTIFIER, NUMBER, KEYWORD, OPERATOR. Regular expressions define the patterns for each token type. A finite automaton DFA or NFA implements the pattern matching. The symbol table stores identifiers and their attributes like type and scope. Lexer tools like Flex generate lexers from regular expression specifications.

## Parsing
Parsing verifies that the token sequence conforms to the language grammar. Context-free grammars CFG define language syntax using production rules. Parse tree shows the hierarchical structure of the input. Top-down parsing starts from the start symbol and tries to derive the input. Recursive descent parsing is a top-down method where each grammar rule has a corresponding function. LL parsers read input Left to right and produce Leftmost derivation. Bottom-up parsing builds the parse tree from leaves to root. LR parsers read Left to right and produce Rightmost derivation in reverse. LALR parsers are the most common bottom-up parsers used in tools like Yacc and Bison.

## Programming Paradigms
Imperative programming describes how to do something using statements that change program state. Procedural programming organizes imperative code into procedures or functions. Object oriented programming organizes code into objects that combine state and behavior. Functional programming treats computation as evaluation of mathematical functions avoiding mutable state and side effects. Functions are first-class values that can be passed and returned. Map filter and reduce are common functional operations. Declarative programming describes what to achieve not how. SQL is declarative. Logic programming expresses facts and rules then queries them, as in Prolog.

## Type Systems
Static typing checks types at compile time. Errors caught early before runtime. Java C and C++ are statically typed. Dynamic typing checks types at runtime. More flexible but errors appear at runtime. Python JavaScript and Ruby are dynamically typed. Strong typing prevents implicit type coercions between incompatible types. Python is strongly typed. Weak typing allows implicit coercions. JavaScript is weakly typed. Type inference allows the compiler to deduce types without explicit annotations. Haskell and modern Java with var use type inference. Garbage collection automatically reclaims memory no longer referenced by the program. Java Python and JavaScript use garbage collection. C and C++ require manual memory management with malloc and free or new and delete.

## Interpreters vs Compilers
A compiler translates the entire source program to machine code before execution. The compiled program runs directly on hardware and is fast. C C++ and Rust are compiled. An interpreter reads and executes source code line by line without producing machine code. Easier to debug and more portable but slower. Python and Ruby use interpreters. Just-in-time JIT compilation compiles code at runtime just before execution combining benefits of both. Java uses JIT via the JVM Java Virtual Machine. Bytecode is an intermediate representation executed by a virtual machine. Java compiles to bytecode run by the JVM. Python compiles to bytecode run by CPython. Virtual machines provide portability since bytecode runs on any platform with the VM installed.
