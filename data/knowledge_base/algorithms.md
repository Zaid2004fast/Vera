# Algorithms

## Sorting Algorithms
Bubble sort repeatedly swaps adjacent elements if they are in the wrong order. Time complexity is O(n squared) in average and worst case and O(n) best case when already sorted. Space is O(1). It is stable. Selection sort finds the minimum element and places it at the beginning repeatedly. Time is O(n squared) in all cases. Space is O(1). It is not stable. Insertion sort builds the sorted array one element at a time by inserting each element into its correct position. Time is O(n squared) average and worst case and O(n) best case. Space is O(1). It is stable and efficient for small or nearly sorted arrays. Merge sort divides the array in half recursively then merges the sorted halves. Time is O(n log n) in all cases. Space is O(n) for the temporary array. It is stable and preferred when stability matters. Quick sort picks a pivot and partitions the array so elements less than pivot are on the left and greater are on the right, then recurses. Average time is O(n log n) and worst case is O(n squared) when pivot is always the smallest or largest element. Space is O(log n) average for the call stack. It is not stable but is the fastest in practice due to cache efficiency. Heap sort builds a max heap then repeatedly extracts the maximum. Time is O(n log n) in all cases. Space is O(1). It is not stable. Counting sort counts occurrences of each element and works only for integers in a known range. Time is O(n plus k) where k is the range. It is stable. Radix sort sorts digit by digit from least to most significant using a stable sort like counting sort as a subroutine. Time is O(d times n plus k) where d is the number of digits. Bucket sort distributes elements into buckets and sorts each bucket. Average case is O(n plus k) but worst case is O(n squared).

## Binary Search
Binary search finds an element in a sorted array by repeatedly halving the search space. Time complexity is O(log n). Space is O(1) for iterative and O(log n) for recursive due to call stack. The array must be sorted. The algorithm compares the target with the middle element and eliminates half the array based on the comparison. Common mistakes include off-by-one errors in setting left and right pointers and integer overflow when computing mid as (left plus right) divided by 2. Safe version uses left plus (right minus left) divided by 2. Binary search can also find the first or last occurrence of a target by modifying the condition to continue searching after a match. This is useful for finding insertion positions or range boundaries.

## Recursion and Dynamic Programming
Recursion solves a problem by breaking it into smaller subproblems of the same type. Every recursive function needs a base case to stop recursion and a recursive case that moves toward the base case. Call stack space is O(depth of recursion). Dynamic programming solves problems by breaking them into overlapping subproblems and storing results to avoid recomputation. Top-down DP or memoization uses recursion with a cache. Bottom-up DP fills a table iteratively. Key DP problems: Fibonacci with memoization runs in O(n) instead of O(2 to the n). Longest common subsequence LCS runs in O(m times n) where m and n are string lengths. 0/1 Knapsack runs in O(n times W) where n is items and W is capacity. Longest increasing subsequence LIS runs in O(n squared) with basic DP and O(n log n) with patience sorting. Coin change minimum coins runs in O(amount times number of coins). Matrix chain multiplication runs in O(n cubed). Edit distance Levenshtein runs in O(m times n). Identifying DP: if the problem asks for maximum minimum count or existence and has optimal substructure and overlapping subproblems it is likely DP.

## Graph Algorithms
Depth first search DFS explores as deep as possible before backtracking. Uses a stack or recursion. Time is O(V plus E). Used for topological sort, cycle detection, connected components, and maze solving. Breadth first search BFS explores all neighbors before going deeper. Uses a queue. Time is O(V plus E). Finds shortest path in unweighted graphs. Used for level-order traversal, bipartite checking, and shortest path. Dijkstra shortest path algorithm works on graphs with non-negative weights. Uses a min-heap priority queue. Time is O((V plus E) log V). Does not work with negative weights. Bellman-Ford handles negative weight edges and detects negative weight cycles. Time is O(VE). Floyd-Warshall finds all-pairs shortest paths in O(V cubed) using dynamic programming. Minimum spanning tree: Kruskal sorts edges by weight and adds them if they do not create a cycle using Union-Find. Time is O(E log E). Prim starts from a node and greedily adds the cheapest edge connecting the tree to a new node using a priority queue. Time is O((V plus E) log V). Topological sort orders vertices in a DAG such that every directed edge goes from earlier to later. Kahn algorithm uses BFS with in-degree tracking. DFS-based version records finish times. Used in build systems, task scheduling, and dependency resolution. Union-Find or Disjoint Set Union DSU supports find and union operations. Path compression and union by rank give nearly O(1) amortized per operation. Used in Kruskal and cycle detection.

## Sliding Window and Two Pointers
Sliding window maintains a window of elements and slides it across the array. Used for maximum sum subarray of size k in O(n), longest substring without repeating characters in O(n), and minimum window substring. Two pointers use two indices moving toward each other or in the same direction. Used for two sum on sorted array in O(n), removing duplicates from sorted array in O(n), and container with most water. Both patterns convert O(n squared) brute force solutions to O(n).

## Divide and Conquer
Divide and conquer splits the problem into independent subproblems, solves them recursively, and combines results. Examples: merge sort, quick sort, binary search, and closest pair of points. The master theorem gives the time complexity of divide and conquer recurrences T(n) equals aT(n divided by b) plus f(n). If f(n) is O(n to the log base b of a minus epsilon) then T(n) is Theta(n to the log base b of a). If f(n) is Theta(n to the log base b of a) then T(n) is Theta(n to the log base b of a times log n).
