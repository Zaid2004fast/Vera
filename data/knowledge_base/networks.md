# Computer Networks

## OSI and TCP/IP Models
The OSI model has 7 layers. Physical layer transmits raw bits over a physical medium. Data Link layer handles framing, MAC addressing, and error detection using Ethernet and Wi-Fi. Network layer handles logical addressing and routing using IP. Transport layer provides end-to-end communication using TCP and UDP. Session layer manages sessions between applications. Presentation layer handles data formatting, encryption, and compression. Application layer provides network services to applications using HTTP, FTP, DNS, SMTP. The TCP/IP model has 4 layers: Network Access combining OSI layers 1 and 2, Internet corresponding to OSI Network layer using IP, Transport using TCP and UDP, and Application combining OSI session, presentation, and application layers. Data encapsulation adds headers at each layer going down the stack. Each layer's data unit: application data, TCP segment, IP packet, Ethernet frame.

## TCP vs UDP
TCP Transmission Control Protocol is connection-oriented. It requires a three-way handshake: SYN, SYN-ACK, ACK before data transfer. It provides reliable ordered delivery with error checking, retransmission of lost packets, flow control using sliding window, and congestion control. It is slower due to overhead. Used for HTTP, email, file transfer, and any application needing reliability. UDP User Datagram Protocol is connectionless. It sends packets without establishing a connection or confirming delivery. It is faster with less overhead. No guarantees of delivery, ordering, or duplicate protection. Used for video streaming, gaming, DNS lookups, and VoIP where speed matters more than reliability. TCP connection termination uses a four-way handshake: FIN, ACK, FIN, ACK.

## HTTP and HTTPS
HTTP HyperText Transfer Protocol is an application layer protocol for transferring web resources. It is stateless meaning each request is independent. HTTP methods: GET retrieves a resource and is idempotent and cacheable. POST sends data to create a resource and is not idempotent. PUT replaces a resource and is idempotent. PATCH partially updates a resource. DELETE removes a resource and is idempotent. HTTP status codes: 200 OK success, 201 Created, 301 Moved Permanently redirect, 400 Bad Request client error, 401 Unauthorized, 403 Forbidden, 404 Not Found, 500 Internal Server Error, 503 Service Unavailable. HTTP/1.1 supports persistent connections and pipelining. HTTP/2 multiplexes multiple requests over a single connection and compresses headers improving performance. HTTP/3 uses QUIC protocol over UDP for even lower latency. HTTPS is HTTP over TLS Transport Layer Security. TLS uses asymmetric encryption for key exchange and symmetric encryption for data transfer. It provides confidentiality, integrity, and authentication.

## DNS and IP Addressing
DNS Domain Name System translates domain names to IP addresses. It is a distributed hierarchical database. DNS resolution: browser checks local cache, then OS cache, then queries the recursive resolver, which queries root nameservers, then TLD nameservers, then authoritative nameservers. TTL Time To Live determines how long a DNS record is cached. Common DNS record types: A maps domain to IPv4 address, AAAA maps to IPv6 address, CNAME is an alias for another domain, MX specifies mail servers, TXT stores arbitrary text often for verification. IPv4 addresses are 32-bit numbers in dotted decimal notation like 192.168.1.1. IPv6 addresses are 128-bit in hexadecimal. IPv4 has only 4.3 billion addresses leading to the development of IPv6 and NAT Network Address Translation which allows multiple devices to share one public IP. Subnetting divides a network into smaller subnetworks using a subnet mask. CIDR Classless Inter-Domain Routing notation expresses a subnet as IP address slash prefix length like 192.168.1.0/24 meaning the first 24 bits are the network portion.

## Sockets and Ports
A socket is an endpoint for network communication identified by an IP address and port number. Port numbers range from 0 to 65535. Well-known ports 0 to 1023 are reserved for standard services: 80 HTTP, 443 HTTPS, 22 SSH, 21 FTP, 25 SMTP, 53 DNS, 3306 MySQL, 5432 PostgreSQL. A server binds to a port and listens for incoming connections. A client connects to the server IP and port. TCP socket programming: server calls socket, bind, listen, accept, recv, send, close. Client calls socket, connect, send, recv, close. WebSockets provide full-duplex communication over a single TCP connection allowing the server to push data to the client without the client polling. Used in real-time applications like chat, gaming, and live updates.
