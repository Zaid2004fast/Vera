# Computer Architecture

## CPU Basics
The CPU Central Processing Unit executes instructions. It consists of the ALU Arithmetic Logic Unit which performs arithmetic and logical operations, the Control Unit which decodes instructions and coordinates components, and registers which are small fast storage locations inside the CPU. The program counter PC holds the address of the next instruction to execute. The instruction register IR holds the current instruction being executed. The memory address register MAR holds the address of memory to read or write. The memory data register MDR holds the data read from or written to memory.

## Instruction Cycle
The fetch-decode-execute cycle is the fundamental operation of a CPU. Fetch: the CPU reads the instruction from memory at the address in the program counter and increments the PC. Decode: the control unit interprets the instruction to determine what operation to perform. Execute: the ALU or other units perform the operation. This cycle repeats for every instruction in a program. Modern CPUs use pipelining to overlap these stages for multiple instructions simultaneously.

## Pipelining
Pipelining divides instruction execution into stages and processes multiple instructions simultaneously, one in each stage. A 5-stage pipeline: Fetch, Decode, Execute, Memory Access, Write Back. When the pipeline is full each stage handles a different instruction simultaneously giving near one instruction per cycle throughput. Pipeline hazards: data hazard occurs when an instruction depends on the result of a previous instruction not yet complete. Control hazard occurs with branches when the next instruction to fetch is not yet known. Structural hazard occurs when two instructions need the same hardware resource simultaneously. Solutions include stalling the pipeline, forwarding data between stages, and branch prediction.

## Memory Hierarchy
Memory hierarchy organizes storage by speed and cost. Registers are fastest, smallest, and most expensive — a few hundred bytes inside the CPU. Cache memory L1 L2 L3 is fast SRAM built into or near the CPU, stores frequently used data and instructions. L1 is smallest fastest a few KB per core. L2 is larger slower a few MB. L3 is shared between cores tens of MB. Main memory RAM is slower larger and cheaper, gigabytes, stores running programs and data. Secondary storage SSD or HDD is slowest largest cheapest, terabytes, stores persistent data. Cache hit means data found in cache giving fast access. Cache miss means data must be fetched from slower memory. Cache replacement policies include LRU and FIFO.

## Number Systems
Binary uses base 2 with digits 0 and 1. Each bit position represents a power of 2. Hexadecimal uses base 16 with digits 0-9 and A-F. Each hex digit represents 4 binary bits making it compact for representing binary data. Conversion: binary to decimal sum the powers of 2 for each 1 bit. Decimal to binary repeatedly divide by 2 and collect remainders. Two's complement represents signed integers. Positive numbers are same as unsigned binary. Negative numbers: invert all bits and add 1. An 8-bit two's complement number represents -128 to 127. Overflow occurs when arithmetic result exceeds the range representable by the number of bits.

## Input Output
IO devices communicate with the CPU through IO controllers or device drivers. Programmed IO: the CPU polls the device status register repeatedly waiting for IO to complete, wasting CPU cycles. Interrupt-driven IO: the device sends an interrupt when IO completes, freeing the CPU to do other work until interrupted. DMA Direct Memory Access: a DMA controller transfers data directly between device and memory without involving the CPU for each byte, only interrupting when the entire transfer is complete. IO ports are addresses used to communicate with IO devices. Memory-mapped IO maps device registers into the memory address space allowing the CPU to read and write devices using normal memory instructions.
