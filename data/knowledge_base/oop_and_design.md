# Object Oriented Programming and Design

## OOP Pillars
Encapsulation bundles data and methods that operate on the data within a class and restricts direct access to some components using access modifiers. Private members are accessible only within the class. Protected members are accessible within the class and subclasses. Public members are accessible from anywhere. Benefits include data hiding, reducing complexity, and making code easier to maintain. Abstraction hides complex implementation details and exposes only necessary functionality through interfaces or abstract classes. It reduces complexity and allows implementation to change without affecting users. Inheritance allows a class to derive properties and behaviors from another class. The child class inherits all public and protected members of the parent. Promotes code reuse. Types: single, multiple in C++ but not Java, multilevel, hierarchical. Composition is often preferred over deep inheritance hierarchies because it is more flexible. Polymorphism allows objects of different types to be treated as objects of a common type. Compile-time or static polymorphism uses method overloading where multiple methods have the same name but different parameters. Runtime or dynamic polymorphism uses method overriding where a child class provides a specific implementation of a parent class method. Virtual functions and dynamic dispatch implement runtime polymorphism.

## SOLID Principles
Single Responsibility Principle: a class should have only one reason to change meaning it should have only one job or responsibility. Open Closed Principle: software entities should be open for extension but closed for modification. Add new functionality by extending existing code not changing it. Liskov Substitution Principle: objects of a subclass should be replaceable with objects of the parent class without breaking the application. Subclasses should not weaken preconditions or strengthen postconditions. Interface Segregation Principle: clients should not be forced to depend on interfaces they do not use. Prefer many specific interfaces over one general interface. Dependency Inversion Principle: high-level modules should not depend on low-level modules, both should depend on abstractions. Depend on interfaces or abstract classes not concrete implementations.

## Design Patterns
Creational patterns deal with object creation. Singleton ensures only one instance of a class exists and provides a global access point. Implemented with a private constructor and a static instance. Not thread-safe without synchronization. Factory Method defines an interface for creating an object but lets subclasses decide which class to instantiate. Abstract Factory creates families of related objects. Builder constructs complex objects step by step separating construction from representation. Prototype creates new objects by cloning existing ones. Structural patterns deal with object composition. Adapter converts the interface of one class to another expected by the client. Decorator adds behavior to objects dynamically without modifying their class by wrapping them. Observer defines a one-to-many dependency so that when one object changes state all dependents are notified automatically. Used in event systems and MVC. Strategy defines a family of algorithms, encapsulates each one, and makes them interchangeable. The client can choose the algorithm at runtime. Command encapsulates a request as an object allowing parameterization and queuing. MVC Model View Controller separates application into three components. Model holds data and business logic. View displays data to the user. Controller handles user input and updates model and view.

## Classes and Objects
A class is a blueprint for objects defining attributes and methods. An object is an instance of a class allocated in heap memory. Constructor is called when an object is created and initializes its state. Destructor is called when an object is destroyed and releases resources. In Python __init__ is the constructor and __del__ is the destructor. In Java the garbage collector handles memory deallocation automatically. Static members belong to the class not instances. There is only one copy shared by all instances. Abstract class cannot be instantiated and may have abstract methods that must be overridden by subclasses. An interface defines a contract of methods that implementing classes must provide. In Java a class can implement multiple interfaces but extend only one class. In Python multiple inheritance is supported directly. Method overloading defines multiple methods with the same name but different parameter types or counts. Python does not support true overloading but achieves it with default arguments. Method overriding provides a new implementation for a method inherited from a parent class using the same signature.
