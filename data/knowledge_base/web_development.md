# Web Development

## How the Web Works
When you type a URL in a browser: DNS resolves the domain to an IP address. Browser sends an HTTP request to the server. Server processes the request and sends back an HTTP response with status code and body. Browser renders the HTML CSS and JavaScript. HTTP is stateless meaning each request is independent with no memory of previous requests. Cookies and sessions add statefulness. URL Uniform Resource Locator has components: protocol, domain, port, path, query parameters, fragment. HTTP methods: GET retrieves data, POST creates data, PUT updates data, DELETE removes data, PATCH partially updates. HTTP status codes: 200 OK, 201 Created, 301 Redirect, 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 500 Server Error.

## HTML and CSS
HTML HyperText Markup Language defines structure and content using elements and tags. Semantic HTML uses meaningful tags like header nav main article footer instead of generic divs improving accessibility and SEO. The DOM Document Object Model is a tree representation of the HTML document that JavaScript can manipulate. CSS Cascading Style Sheets controls visual presentation. Selectors target elements: element selectors, class selectors with dot, ID selectors with hash. Specificity determines which CSS rule wins when multiple rules target the same element. Box model: every element is a box with content padding border and margin. Flexbox is a one-dimensional layout system for arranging items in rows or columns. CSS Grid is a two-dimensional layout system for complex page layouts. Responsive design uses media queries to adapt layout to different screen sizes.

## JavaScript
JavaScript is a dynamically typed interpreted language for web interactivity. Variables: var has function scope and is hoisted. let and const have block scope and are preferred. const prevents reassignment not mutation. Functions can be declared with function keyword or as arrow functions. Closures: a function retains access to variables from its outer scope even after the outer function returns. Promises handle asynchronous operations. A promise is either pending, fulfilled, or rejected. Async and await syntax makes promise-based code look synchronous. The event loop processes asynchronous callbacks after the call stack is empty. DOM manipulation: document.querySelector selects elements, addEventListener attaches event handlers, innerHTML sets HTML content. Fetch API makes HTTP requests returning promises.

## REST APIs
REST Representational State Transfer is an architectural style for building APIs over HTTP. Stateless: each request contains all information needed. Resources are identified by URLs. HTTP methods map to CRUD operations: GET for read, POST for create, PUT for update, DELETE for delete. JSON JavaScript Object Notation is the standard data format for REST APIs. Request headers include Content-Type application/json and Authorization for authentication. Response includes status code and JSON body. API versioning with URL prefix like /api/v1 allows changes without breaking existing clients. Rate limiting controls how many requests a client can make per time period. Pagination handles large result sets using limit and offset or cursor-based pagination.

## Databases in Web Development
Relational databases like PostgreSQL and MySQL store structured data in tables with relationships. ORMs Object Relational Mappers like SQLAlchemy and Django ORM let you interact with databases using Python objects instead of raw SQL. Connection pooling reuses database connections to avoid the overhead of creating new connections for each request. Indexing frequently queried columns speeds up reads. Migrations are version-controlled changes to the database schema. NoSQL databases like MongoDB store documents as JSON-like objects without a fixed schema. Redis is an in-memory key-value store used for caching session storage and message brokering. Environment variables store database credentials securely outside the codebase.

## Frontend Frameworks
React is a JavaScript library for building user interfaces using components. Each component is a reusable piece of UI that manages its own state. JSX combines HTML and JavaScript in one syntax. useState hook manages local component state. useEffect hook handles side effects like fetching data. Virtual DOM diffs the previous and new state and updates only changed parts of the real DOM efficiently. Props pass data from parent to child components. State lifting moves shared state to the common parent. Vue is a progressive framework similar to React with a gentler learning curve. Angular is a full framework by Google with built-in tools for routing forms and HTTP. Single page applications SPAs load once and dynamically update content without full page reloads.
